How Federated Learning Works
Privacy-preserving architecture
Step 1: Local Training
Each client trains on its local network data only. Raw packets never leave the node.
Step 2: Share Model Updates
Only encrypted gradient updates (not data) are sent to the central server.
Step 3: FedAvg Aggregation
Server aggregates updates using Federated Averaging to build a stronger global model.
Step 4: Global Model Distribution
Updated global model is pushed back to all clients. Privacy preserved throughout.
